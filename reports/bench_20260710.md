# sorohunter — benchmark report

**Precision 100% · Recall 100%** (tp 3, fp 0, fn 0)

An ABI-driven adversarial hunter, run in local fork-sim over a benchmark corpus of Soroban contracts with planted missing-auth bugs and clean decoys. Each function is invoked under empty authorization; a call that changes state (emits an event) without a signature is a missing-auth finding, with the invocation itself as the executed PoC.

## Probes (executed, with verdict)

### vuln_vault
- **[view] balance(address)** — succeeded, no event — read-only
- **[held] deposit(address,i128)** — aborted under empty auth
- **[BREACH] withdraw(address,i128)** — succeeded and emitted an event under empty auth — state change without a signature

### safe_vault
- **[view] balance(address)** — succeeded, no event — read-only
- **[held] deposit(address,i128)** — aborted under empty auth
- **[held] withdraw(address,i128)** — aborted under empty auth

### chain_vault
- **[view] pot()** — succeeded, no event — read-only
- **[held] withdraw(i128)** — aborted under empty auth
- **[view] set_admin(address)** — succeeded, no event — read-only
- **[CHAIN] set_admin->withdraw()** — foothold set_admin() under empty auth seized control, unlocking gated withdraw() for the attacker — composition, PoC is the executed sequence

### safe_chain_vault
- **[view] pot()** — succeeded, no event — read-only
- **[held] withdraw(i128)** — aborted under empty auth
- **[held] set_admin(address)** — aborted under empty auth

### upgrade_vault
- **[view] ping()** — succeeded, no event — read-only
- **[held] upgrade(bytes_n:32)** — aborted under empty auth
- **[HIJACK] upgrade(bytes_n:32)** — upgrade() swapped the contract code under empty auth — the attacker payload now executes (pwned=1337): OBJ-SEIZE, arbitrary control

### safe_upgrade_vault
- **[view] ping()** — succeeded, no event — read-only
- **[held] upgrade(bytes_n:32)** — aborted under empty auth

## Reading

The hunter caught every planted missing-auth bug and raised no false alarms on the clean decoys. It also does not flag read-only views. This is the precision that makes it safe to point at real public contracts: a false positive on a live protocol would burn the exact credibility this is for.
